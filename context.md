# Context for AI-Driven Executive Command Center (NK Protein)

## Project Overview
Goal: A one-day POC proving that we can process 2 years of historical data to provide "Prescriptive" (action-oriented) advice to the CMD of NK Protein.
The POC replaces static reporting with an AI-first conversational interface that predicts risks and identifies growth opportunities.

## Data Layer
- 24 months (730 days) of historical data.
- 4 CSV files generated by `data-gen.py`:
  - `historical_sales.csv`: Daily sales (10 products), seasonality spikes, growth trend for "Tirupati Cottonseed Oil", anomaly drop for "Olixir Olive Oil".
  - `receivables_ledger.csv`: Invoices, ₹70,00,000 unpaid from "Gujarat Supermarkets" (60-90 days old).
  - `inventory_snapshots.csv`: Weekly stock, "De Oiled Cake" stuck for 120 days.
  - `gst_compliance_log.csv`: Monthly records, ₹15,00,000 mismatch in recent month.

## Architecture
- **Analytics Engine (Python/FastAPI)**: Performs real math (Scikit-Learn/Pandas) for forecasting and risk scoring. Analyzes data, formats raw math, and passes to LLM.
- **LLM Orchestrator**: Transforms raw mathematical output into an executive summary starting with "Accessing SAP Joule...".
- **Frontend (SolidJS + TailwindCSS)**: Professional "Pulse" view (Chart.js) showing 2-year history and 90-day forecast. Command Chat sidebar with critical alerts and action buttons.

## Code Conventions
- **Naming Conventions**: Keep file names lowercase and hyphenated (`data-gen.py`, `dashboard.tsx`, `pulse-chart.tsx`).
- **DRY Concepts**: Do not repeat logic. Extract common utilities.
- **Small Components/Classes/Functions**: Refactor code into small, single-responsibility units instead of long code files.
- **Design Patterns**: Use appropriate design patterns for frontend state management and backend architecture.
